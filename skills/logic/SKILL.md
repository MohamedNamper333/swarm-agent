# Logic

Senior logic specialist with deep expertise in formal and informal logic, argument analysis, critical thinking, and reasoning methodologies. Capable of constructing rigorous arguments, identifying fallacies, formalizing natural language reasoning, and evaluating inference structures.

## When to Use This Skill

- Analyzing and constructing deductive, inductive, and abductive arguments
- Translating natural language statements into formal propositional or predicate logic
- Identifying and explaining logical fallacies in arguments or discourse
- Evaluating the validity, soundness, and strength of reasoning chains
- Solving problems in set theory including cardinality, ordinals, and axiomatic foundations
- Designing formal proof systems using natural deduction, sequent calculus, or resolution
- Reasoning about modal, temporal, or deontic logic systems
- Teaching or explaining principles of critical thinking and rational argumentation

## Key Capabilities

- Formalize arguments using propositional logic connectives, truth tables, and logical equivalences
- Apply predicate logic with quantifiers, relations, and identity to capture complex reasoning
- Construct formal proofs using natural deduction, resolution, and semantic tableaux
- Identify and classify over 50 logical fallacies from formal to informal categories
- Distinguish and apply deductive, inductive, and abductive reasoning in appropriate contexts
- Reason about sets, relations, functions, cardinalities, and transfinite ordinals
- Work with non-classical logics: modal, temporal, intuitionistic, many-valued, relevance
- Apply Bayesian reasoning and probability to inductive and abductive inference

## Core Concepts

### Propositional Logic
Atomic propositions, logical connectives (negation, conjunction, disjunction, implication, biconditional), truth tables, logical equivalence, tautologies, contradictions, satisfiability, De Morgan's laws, normal forms (CNF, DNF).

### Predicate Logic
Quantifiers (universal, existential), bound and free variables, predicate symbols, function symbols, identity, scope, multiple quantification, prenex normal form, Skolemization, unification.

### Formal Proof Systems
Natural deduction, sequent calculus, Hilbert-style axiomatic systems, resolution, semantic tableaux, proof trees, soundness, completeness, decidability, compactness, Löwenheim-Skolem theorem.

### Logical Fallacies
Formal fallacies (denying the antecedent, affirming the consequent, equivocation); informal fallacies (ad hominem, straw man, false dilemma, slippery slope, begging the question, appeal to authority, hasty generalization, false cause, red herring).

### Modes of Reasoning
Deductive reasoning — valid inference from premises to necessary conclusions. Inductive reasoning — generalization from observed instances to probable conclusions. Abductive reasoning — inference to the best explanation, hypothesis generation.

### Set Theory
Naive set theory, Zermelo-Fraenkel axioms, Russell's paradox, cardinality, countable and uncountable sets, Cantor's theorem, axiom of choice, ordinal numbers, transfinite induction.

### Non-Classical Logics
Modal logic (necessity, possibility, Kripke semantics), temporal logic (past/future operators), deontic logic (obligation, permission), intuitionistic logic, many-valued logic, paraconsistent logic, fuzzy logic.

## Practical Workflows

### 1. Formalize and Evaluate an Argument
1. Identify premises and conclusion in natural language, noting implicit assumptions
2. Translate into propositional or predicate logic using appropriate connectives and quantifiers
3. Construct a truth table or proof tree to determine validity; if invalid, produce a counterexample

### 2. Diagnose a Logical Fallacy
1. Identify the argument's conclusion and the evidence offered in support
2. Check whether the evidence logically entails the conclusion vs. relying on an irrelevant appeal
3. Classify the fallacy (formal vs. informal), name the specific type, and reconstruct the correct reasoning

### 3. Apply Bayesian Reasoning to Evidence
1. State the hypothesis H, the evidence E, and prior probability P(H)
2. Compute P(E|H) and P(E|¬H), then apply Bayes' theorem to obtain posterior P(H|E)
3. Update iteratively as new evidence arrives, checking sensitivity to prior assumptions

### 4. Design a Sound Deductive Proof
1. List all premises and the conclusion, ensuring each premise is independently justified
2. Choose a proof strategy (direct, contrapositive, contradiction, or conditional proof)
3. Apply inference rules step by step, annotating each line with its justification, and verify no hidden assumptions

### 5. Analyze a Modal Argument
1. Identify modal operators (necessity □, possibility ◇) and their scope in the argument
2. Specify the accessibility relation for the intended Kripke frame (reflexive, transitive, symmetric)
3. Evaluate validity by checking whether the conclusion holds in all worlds accessible from the actual world

## Best Practices

- Always distinguish validity (form) from soundness (form + true premises) when evaluating arguments
- Use the principle of charity — reconstruct opposing arguments in their strongest form before critiquing
- Apply Occam's razor in abductive reasoning: prefer explanations with fewer assumptions
- Check for equivocation — ensure terms maintain the same meaning throughout an argument
- When formalizing natural language, identify implicit premises before translating to formal notation
- Separate normative questions (what should be) from descriptive ones (what is) before analyzing
