## Instructions

Guide the founder through the complete zero→company lifecycle in 7 phases. Each phase has gates — do not proceed until current gate criteria are met.

### Phase 1: Problem Validation (Weeks 1-2)
- Define the target customer with specific demographics, job-to-be-done, and current workaround
- Conduct 10+ customer interviews using the Problem Validation Interview Protocol
- Validate willingness-to-pay and purchase authority
- Gate: 3+ interviewees explicitly say "I need this" and "I'd pay"

### Phase 2: Solution Scoping (Weeks 2-3)
- Define MVP scope — single feature, single channel, single user type
- Write one-page PRD: user story, success criteria, non-goals
- Map the user flow from sign-up to first value received
- Estimate build time (target: 2-6 weeks for first version)
- Gate: PRD is narrower than you want, specific enough to build from

### Phase 3: MVP Build & Ship (Weeks 3-8)
- Choose stack based on: founder skills, speed to market, maintenance cost
- Build with manual processes before automation (do things that don't scale)
- Ship to first 5 users manually (invite, onboard, support personally)
- Measure: time-to-first-value, completion rate of core action
- Gate: 5 users completed the core action and got value

### Phase 4: PMF Search (Weeks 8-16)
- Track: retention (DAU/MAU > 20%), organic sign-ups, NPS > 30
- Run retention loop experiments: email, feature adoption, onboarding improvements
- Conduct ongoing user interviews weekly
- Identify the top 3 reasons users stay or leave
- Gate: Week-over-week retention for 4 weeks, organic referrals starting

### Phase 5: Pricing & Packaging (Week 12+)
- Research competitor pricing
- Run willingness-to-pay survey with existing users
- Define pricing tiers (free/paid, per-seat/usage/flat)
- Set up Stripe/billing, add paywall to core feature
- Gate: 5+ paid users at projected price point

### Phase 6: Go-To-Market (Week 16+)
- Define positioning: one-liner, elevator pitch, tagline
- Pick 2 channels: content, ads, sales, partnerships, community
- Create landing page with clear value prop and CTA
- Build waitlist or early-access funnel
- Gate: 50+ signups, 10+ active weekly users, 3+ paying

### Phase 7: Scale (Month 6+)
- Hire first employee (customer-facing role)
- Automate manual processes (onboarding, billing, support)
- Set up metrics dashboard (revenue, churn, CAC, LTV)
- Establish company legal and compliance
- Gate: $1K+ MRR, <10% monthly churn, LTV > 3x CAC

## Cross-Cutting Principles
- Talk to users every single week, no exceptions
- Ship continuously — if feature takes >2 weeks, break it down
- Manual first, API later, automation last
- Revenue proves PMF better than any metric
- Avoid premature optimization in code, team, or process

## When NOT to use
- User has an existing, scaling business (use specific growth/scale skills)
- User just wants code written (use senior-developer instead)
- Idea is purely for learning, not building a business

## Return
- Current phase status and gate assessment
- Specific next action to advance past the current gate
- Risk indicators: what could kill the idea in this phase
- Recommended skills from the swarm to execute the next step
