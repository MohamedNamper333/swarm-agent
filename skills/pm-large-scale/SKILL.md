# Large Scale Program Management

## Program vs Project
- **Project**: single deliverable, defined timeline, one team
- **Program**: multiple related projects, strategic outcomes, ongoing
- **Portfolio**: all programs/projects in an organization, resource optimization

## SAFe (Scaled Agile Framework) Essentials
- **Team level**: Agile teams (5-9 people) doing Scrum/Kanban
- **Program level**: Agile Release Train (ART) — 50-125 people, 8-12 teams
- **Large Solution level**: multiple ARTs working together
- **Portfolio level**: strategic alignment, Lean budgeting

## Multi-Team Coordination
- **Scrum of Scrums**: daily sync between team representatives
- **PI Planning**: Program Increment planning every 8-12 weeks
- **Feature Teams** over component teams (end-to-end ownership)
- **System Demo**: integrated demo every iteration (not just team demos)

## Dependency Management
- Track cross-team dependencies clearly
- Mark dependencies on roadmaps
- Have dependency owners resolution meetings
- Use colored flags: Red (blocking), Yellow (risk), Green (on track)

## Governance
- SteerCo meetings: monthly, decisions + escalations
- Stage gates: phase transitions require approval
- RAID log: Risks, Assumptions, Issues, Dependencies
- Rolling wave planning: detailed near-term, high-level far-term

## Communication at Scale
- Weekly newsletters / status reports
- Centralized wiki / Sharepoint / Confluence
- Clear escalation paths
- Distributed decision-making: push decisions to the lowest capable level
- Manage by exception — only escalate what's off track
